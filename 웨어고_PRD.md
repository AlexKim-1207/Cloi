# PRD: 클로이(Cloi) — 패션 이미지 분석 및 유사 상품 추천 토스 미니앱

**작성일**: 2026-04-10  
**상태**: Draft v1.0  
**플랫폼**: 토스 미니앱 (앱인토스 SDK)  
**기술 스택**: React + TypeScript, 앱인토스 SDK, Gemini Flash API (gemini-2.0-flash), 네이버 쇼핑 API

---

## Problem Statement

인스타그램에서 마음에 드는 옷을 발견해도 "이 옷 어디서 사지?"라는 문제에 막혀 구매로 이어지지 않는 경우가 많다. 2030 여성 유저들은 패션 콘텐츠를 소비하면서도 실제 구매 가능한 상품을 찾는 과정(역검색, 브랜드 검색, 직접 문의 등)이 번거롭고 시간이 많이 걸린다는 불편함을 느낀다. 이 문제를 해결하지 못하면 구매 의향이 있는 유저가 이탈하고 패션 플랫폼들은 잠재 고객을 놓치게 된다.

---

## Goals

1. **발견에서 구매까지의 시간 단축**: 이미지 입력 후 유사 상품 결과 확인까지 30초 이내 달성 (런치 후 1개월 기준)
2. **유저 이탈율 감소**: 결과 페이지에서 외부 쇼핑몰 클릭전환율(CTR) 30% 이상 달성
3. **재사용율 확보**: 월간 활성 유저(MAU) 중 2회 이상 사용 비율 40% 이상
4. **AI 분석 정확도**: Gemini 키워드 추출 기반 상품 결과 만족도(별점 또는 thumbs up) 70% 이상
5. **토스 미니앱 내 유입 확대**: 첫 달 내 토스 미니앱 검색 노출을 통한 신규 유저 1만 명 확보

---

## Non-Goals

1. **자체 쇼핑몰 운영**: 상품 판매 및 결제는 무신사·에이블리·지그재그 등 기존 플랫폼에 위임. 직접 커머스 기능은 v1 범위 밖.
2. **인스타그램 외 SNS 지원**: TikTok, Pinterest, 유튜브 링크 지원은 v2에서 검토. v1은 Instagram에 집중.
3. **개인 옷장 관리(outfit log)**: 즐겨찾기 이상의 코디 기록·공유 기능은 별도 피처로 분리.
4. **실시간 재고 정보 제공**: 재고 수량·사이즈 가용 여부는 네이버 쇼핑 API 응답에 의존하며, 자체 재고 연동은 미포함.
5. **브랜드/셀러 광고 상품 노출**: 유료 광고 인벤토리 운영은 수익화 검토 단계(v3 이후)로 미룸.

## User Stories

### 주요 페르소나: 인스타그램 패션 탐색 유저 (2030 여성)

**이미지 입력 및 분석**

- 인스타그램 탐색 중인 유저로서, 마음에 드는 옷 게시물의 링크를 앱에 붙여넣으면 별도 저장 없이 바로 비슷한 상품을 찾고 싶다. 그래야 탐색 흐름이 끊기지 않기 때문이다.
- 인스타그램 링크가 없는 경우, 스크린샷이나 사진을 직접 업로드해서 동일한 분석을 받고 싶다. 링크보다 이미지가 손에 있는 경우가 더 많기 때문이다.
- 앱을 열 때 클립보드에 인스타그램 링크가 복사돼 있으면 자동으로 감지해서 바로 분석을 시작할 수 있기를 바란다. 매번 수동으로 붙여넣는 과정이 번거롭기 때문이다.

**결과 탐색 및 구매 연결**

- 분석 결과로 상품 카드 목록이 나오면 이미지·가격·상품명을 한눈에 보고 싶다. 텍스트만으로는 비교하기 어렵기 때문이다.
- 마음에 드는 상품 카드를 탭하면 무신사, 에이블리, 지그재그 등 구매 가능한 플랫폼으로 바로 이동하고 싶다. 검색창에 상품명을 다시 치는 과정이 불필요하기 때문이다.
- 결과 페이지에서 "이 옷과 비슷한 옷 더 보기" 기능을 통해 추가 탐색을 계속하고 싶다. 첫 결과가 마음에 들지 않을 수도 있기 때문이다.

**엣지 케이스**

- 분석이 실패하거나 관련 상품을 찾지 못했을 때, 막연한 오류 대신 "어떤 부분이 문제인지(예: 이미지 품질 낮음, 옷이 불명확함)"를 알려주는 안내 메시지를 보고 싶다.
- 인스타그램 링크가 비공개 계정 게시물이어서 이미지를 가져올 수 없을 때, 직접 이미지 업로드로 대체 안내를 받고 싶다.

## Requirements

### Must-Have (P0)

| # | 요구사항 | 인수 기준 |
|---|---------|---------|
| P0-1 | **이미지 업로드** | 갤러리에서 이미지 선택 → 업로드 → 제미나이 분석 요청까지 파이프라인 완성. JPG/PNG/WEBP 지원, 최대 10MB |
| P0-2 | **인스타그램 링크 입력** | 공개 계정 게시물 URL 입력 시 이미지 추출 후 분석. 비공개 계정은 에러 메시지 + 이미지 업로드 유도 |
| P0-3 | **클립보드 자동 감지** | 앱 포그라운드 진입 시 클립보드에 instagram.com URL 포함 여부 확인 → 배너/팝업으로 자동 분석 제안 |
| P0-4 | **Gemini 키워드 추출** | 옷 색상·실루엣·소재감·스타일 등 최소 5개 키워드 추출. 응답 5초 이내 (타임아웃 시 재시도 1회) |
| P0-5 | **네이버 쇼핑 API 검색** | 추출된 키워드로 상품 검색. 최소 6개 결과 카드 노출 |
| P0-6 | **상품 카드 UI** | 상품 이미지·이름·가격·플랫폼 배지(무신사/에이블리/지그재그) 포함 카드 컴포넌트 |
| P0-7 | **외부 구매 링크 연결** | 카드 탭 시 해당 상품 페이지로 외부 브라우저 또는 앱 딥링크 이동 |
| P0-8 | **로딩 및 에러 처리** | 분석 중 로딩 스피너 + 단계 표시("옷 분석 중…", "상품 찾는 중…"). API 실패 시 재시도 버튼 포함 에러 화면 |

### Nice-to-Have (P1)

| # | 요구사항 | 인수 기준 |
|---|---------|---------|
| P1-1 | **결과 필터링** | 가격대·플랫폼별 필터 적용 가능. 필터 변경 시 결과 즉시 업데이트 |
| P1-2 | **상품 즐겨찾기** | 하트 버튼으로 찜 목록 저장. 토스 계정 기반 또는 로컬 스토리지 |
| P1-3 | **분석 이력** | 최근 분석한 이미지 및 결과 3~5건 히스토리 저장 및 재조회 |
| P1-4 | **공유하기** | 분석 결과 링크 또는 이미지 카드를 카카오톡·인스타그램으로 공유 |
| P1-5 | **결과 피드백** | 결과 만족도 thumbs up/down 수집. 낮은 평점 시 "키워드 직접 수정" 옵션 제공 |

### Future Considerations (P2)

| # | 요구사항 |
|---|---------|
| P2-1 | Pinterest, TikTok 등 타 SNS 링크 지원 |
| P2-2 | 옷 외 액세서리·신발 카테고리 확장 |
| P2-3 | 코디 전체 이미지에서 아이템별 분리 분석 ("상의만 찾기" / "하의만 찾기") |
| P2-4 | 유사도 기반 랭킹 고도화 (벡터 임베딩 비교) |
| P2-5 | 브랜드 파트너십 기반 수익화 (네이티브 광고, 제휴 커미션) |

## Success Metrics

### Leading Indicators (런치 후 1~4주)

| 지표 | 목표 | 측정 방법 |
|-----|------|---------|
| 분석 완료율 | 입력 시작 대비 결과 수신 80% 이상 | 분석 요청 이벤트 vs 결과 렌더링 이벤트 |
| 결과 CTR (외부 링크 클릭) | 결과 페이지 진입 대비 30% 이상 | 카드 탭 이벤트 / 결과 페이지 진입 이벤트 |
| API 응답 시간 | Gemini + 네이버 쇼핑 합산 10초 이내 P95 | 서버사이드 latency 로그 |
| 에러율 | 전체 분석 요청 중 5% 미만 | API 실패 이벤트 카운트 |

### Lagging Indicators (런치 후 1~3개월)

| 지표 | 목표 | 측정 방법 |
|-----|------|---------|
| MAU 재사용율 (2회 이상) | 40% 이상 | 유저별 세션 카운트 |
| 결과 만족도 | thumbs up 70% 이상 | 피드백 이벤트 집계 |
| 신규 유저 유입 | 첫 달 1만 명 | 앱인토스 SDK 유저 등록 수 |
| 토스 미니앱 평점 | 4.0 이상 | 앱스토어/토스 마켓 평점 |
